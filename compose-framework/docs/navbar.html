Navigation Bar

You can create a responsive navigation by setting a class of cdk-navbar to a <nav> element, and setting a class of cdk-nav-links-list to an <ul> inside the <nav>. The navigation will span 100% of its container. Links will line horizontally, and wrap when the screen shrinks.

If you wish to set an active link, you can do so by adding a class of cdk-nav-link-active to an <a> within cdk-nav-links-list.

Optionally, you can add a text-based logo to the navigation by setting a class of cdk-text-logo to a text element. This will give your selected element a font-size equivalent to an <h6> element. Additionally, if you place an <a> within cdk-text-logo, the element will look like regular text, and not link text. 

<nav class="cdk-navbar">
    <p class="cdk-logo-text"><a class="cdk-logo-link" href="#">Logo Example</a></p>
    <ul class="cdk-nav-links-list">
        <li><a class="cdk-nav-link-active" href="#">Active Nav Link</a></li>
        <li><a href="#">Nav Link</a></li>
        <li><a href="#">Nav Link</a></li>
         <li><a href="#">Nav Link</a></li>
    </ul>
</nav>

Hamburger Navigation

Compose comes with a CSS only hamburger navigation. The same navigation is being used for this documentation page. Check it out! Simply reduce the size of your screen or open this website in a mobile device!

If you wish your navigation to collapse into a hamburger element on mobile screens you can wrap the elements you need to hide within an element with an id of cdk-ham-navbar. Outside cdk-ham-navbar, create an <a> with an id of cdk-ham-nav-toggle and href="#cdk-ham-navbar". The cdk-ham-nav-toggle id will create a button no wider than 4rem. Inside the toggle button you may place any short text or small image you wish the user to see (something descriptive such as the word “Menu” is recommended).

To add close options to the hamburger navigation, you have two options. You can choose to combine these or only add one.

  1. Create an <li> within cdk-nav-links-list, with a class of cdk-ham-nav-close. Within that item add an <a>with href="#cdk-ham-nav-toggle". Inside the <a> you may place text or image you wish the user to see (an “x” or the word “close” is recommended).
  2. Create an empty <a> with a class of cdk-ham-nav-overlay, and href="#cdk-ham-nav-toggle", at the bottom of the cdk-ham-navbar element. This will create a clickable/tappable opaque overlay behind the navigation, when it is expanded. 
<nav class="cdk-navbar">
    <p class="cdk-logo-text"><a class="cdk-logo-link" href="#">Logo Example</a></p>
    
    <!--Toggle button-->
    <a id="cdk-ham-nav-toggle" href="#cdk-ham-navbar" aria-label="Open Main Menu">Menu</a>
     
    <!--Anything within #cdk-ham-navbar will be hidden in mobile unless toggled-->
    <div id="cdk-ham-navbar">
  
        <ul class="cdk-nav-links-list">
            <!--Close button-->
            <li class="cdk-ham-nav-close">
                <a aria-label="Close Main Menu" href="#cdk-nav-toggle">X</a>
            </li>
        
            <!--Navigation Links-->
            <li><a class="cdk-nav-link-active" href="#">Active Nav Link</a></li>
            <li><a href="#">Nav Link</a></li>
            <li><a href="#">Nav Link</a></li>
            <li><a href="#">Nav Link</a></li>
        </ul>

        <!--Close overlay-->
        <a href="#cdk-ham-nav-toggle" aria-label="Close Main Menu" class="cdk-ham-nav-overlay"></a>
    </div>
</nav>

Note on Accessibility

To help with screen reader accessibility, add aria-label="Close Main Menu" to any menu-closing <a> and aria-label="Open Main Menu" to any menu-opening <a>.

CSS Only Hamburger Nav Caveats

The hamburger navigation works by using the CSS :target pseudo-class. Because of this the hamburger navigation will open whenever an element with href="#cdk-ham-navbar" is clicked/tapped, or when the browser address bar is set to a url ending in #cdk-ham-navbar.
