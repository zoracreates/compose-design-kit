Accessible "Read More" Links

Users of screen readers often have the links on a page read out loud to them. This usually allows them to scan websites more quickly, unless the site has too many non-descriptive links, such as multiple “Read More” links. However, “Read More” links can help some sighted users to scan pages for clickable items. While there are other ways to make items look clickable, sometimes the “Read More” is difficult to avoid. For those difficult situations, Compose created an accessible “Read More” link inspired by Matt Wilcox’s work.

These links allow you to include descriptive text for screen readers, and show “Read More” for sighted users. To use these links simply add the classes cdk-more-link and cdk-default-more-text to an <a>, and include the descriptive screen reader text within a <span> with a class of cdk-sr-only inside the <a>. You can also add an arrow to the end of the link by using an <i> with a class of cdk-arrow and the aria-hidden attribute. 

<a class="cdk-more-link cdk-default-more-text">
    <span class="cdk-sr-only">
        Empty link, screen reader only text example
    </span>
    <i class="cdk-arrow" aria-hidden></i>
</a>

Using a Different More Text

Compose improves on Wilcox’s code by allowing you to use your own custom “More” text. The class cdk-default-more-text places “Read More” within the cdk-more-link. If you wish to use a text other that “Read More” you may omit cdk-default-more-text, and place any text you like within the <a class=”cdk-more-link”> tag. 

<a class="cdk-more-link">
    View Details
    <span class="cdk-sr-only">
        Empty link, screen reader only text example
    </span>
    <i class="cdk-arrow" aria-hidden></i>
</a>

Responsive More Blocks

Matt Willcox also talks about responsive “Read More” links . Sometimes there is a block of content associated with a “Read More” link , and it is a good idea to make this whole element clickable on large screens, and only the “Read More” link clickable on small screens. This allows flexibility on large screens but limits the size of the link in small devices, where the user is more likely to accidentally tap/click a link while scrolling down. To achieve this effect, place the cdk-more-link within the block of content in question, and place a class of cdk-more-block on the block. 

<p class="cdk-more-block">
    Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do
    eiusmod tempor incididunt ut labore et dolore magna aliqua. 
    Ut enim ad minim veniam, quis nostrud exercitation ullamco 
    laboris nisi ut aliquip ex ea commodo consequat. 
    <a class="cdk-more-link cdk-default-more-text">
        <span class="cdk-sr-only">
            Empty link, screen reader only text example
        </span>
        <i class="cdk-arrow" aria-hidden></i>
    </a>
</p>
